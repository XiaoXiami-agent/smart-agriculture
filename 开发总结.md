# 智慧农业小程序开发总结

---

## 开发流程（标准步骤）

```
需求调研 → 结构化文档 → 搭建框架 → 制定开发步骤 → 开发实施
```

### 第一步：调研需求
- 用户访谈、场景分析
- 功能清单梳理
- 竞品分析

### 第二步：生成结构化文档 & 搭建框架
- 编写《需求规格说明书》
- 设计数据库模型
- 搭建项目代码框架
- 定义 API 接口规范

### 第三步：制定开发步骤 & 确认顺序
- 任务拆分与排期
- 确定依赖关系
- 制定里程碑

## 四、遇到的问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 定位权限报错 | `app.json` 缺少声明 | 添加 `requiredPrivateInfos` |
| 逆地理编码失败 | `geo.qweather.com` DNS 解析失败 | 改用直接坐标查询 |
| 外部 API 超时 | 云函数网络环境受限 | 和风 API 支持直接用坐标查天气 |
| 腾讯地图 key 无效 | key 未申请/配置错误 | 绕过逆地理编码方案 |

---

## 五、解决方案

**最终采用：坐标直连天气 API**

```
定位坐标 → 和风天气 API(location=经度,纬度) → 直接返回天气数据
```

- 无需逆地理编码
- 城市显示"未知"（可接受）

### 修改的文件

| 文件 | 修改内容 |
|------|----------|
| `app.json` | 添加定位权限声明 |
| `cloudfunctions/weatherApi/index.js` | 用坐标直接调用天气 API |
| `pages/index/index.js` | 更新定位逻辑，直接显示天气 |
