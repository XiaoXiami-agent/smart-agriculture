# 智慧农业小程序开发总结

---

## 开发流程（标准步骤）

```
需求调研 → 结构化文档 → 搭建框架 → 制定开发步骤 → 开发实施 → 测试验证 → 部署上线
```

### 第一步：调研需求
- 用户访谈、场景分析
- 功能清单梳理
- 竞品分析

### 第二步：生成结构化文档 & 搭建框架
- 编写《需求规格说明书》
- 设计数据库模型
- 搭建项目代码框架
- 定义 API 接口规范

### 第三步：制定开发步骤 & 确认顺序
- 任务拆分与排期
- 确定依赖关系
- 制定里程碑

### 第四步：开发实施
- 按计划开发各个功能模块
- 遵循代码规范
- 及时提交 Git

### 第五步：测试验证 ⭐（重要）
- 单元测试：每个函数/模块独立测试
- 集成测试：模块间协作测试
- 端到端测试：完整业务流程测试
- 边界测试：异常情况、极端数据
- 回归测试：确保新功能不破坏已有功能

### 第六步：部署上线
- 代码审查（Code Review）
- 合并到主分支
- 部署到生产环境
- 监控上线效果

## 四、遇到的问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 定位权限报错 | `app.json` 缺少声明 | 添加 `requiredPrivateInfos` |
| 逆地理编码失败 | `geo.qweather.com` DNS 解析失败 | 改用直接坐标查询 |
| 外部 API 超时 | 云函数网络环境受限 | 和风 API 支持直接用坐标查天气 |
| 腾讯地图 key 无效 | key 未申请/配置错误 | 绕过逆地理编码方案 |

---

## 五、解决方案

**最终采用：坐标直连天气 API**

```
定位坐标 → 和风天气 API(location=经度,纬度) → 直接返回天气数据
```

- 无需逆地理编码
- 城市显示"未知"（可接受）

### 修改的文件

| 文件 | 修改内容 |
|------|----------|
| `app.json` | 添加定位权限声明 |
| `cloudfunctions/weatherApi/index.js` | 用坐标直接调用天气 API |
| `pages/index/index.js` | 更新定位逻辑，直接显示天气 |

### Git 工作流程

```
1. 每次开发新功能前：git pull origin main
2. 创建功能分支：git checkout -b feature/功能名
3. 开发完成后：git add . && git commit -m "feat: 功能描述"
4. 推送分支：git push -u origin feature/功能名
5. 在 GitHub 创建 Pull Request 合并到 main
```

### 功能开发检查清单

每个功能开发完成后，必须验证以下内容：

#### 代码层面
- [ ] 代码无语法错误
- [ ] 无 console.log 调试代码残留
- [ ] 变量/函数命名规范
- [ ] 必要的注释说明

#### 功能层面
- [ ] 正常流程测试通过
- [ ] 边界条件测试通过
- [ ] 异常流程有错误提示
- [ ] 加载状态有 Loading 提示

#### 用户体验
- [ ] 按钮有点击反馈
- [ ] 操作有成功/失败提示
- [ ] 页面跳转流畅
- [ ] 空状态有友好提示

#### 兼容性
- [ ] 在开发者工具正常运行
- [ ] 在真机测试正常运行（定位等功能必须真机）
- [ ] 不同屏幕尺寸适配
